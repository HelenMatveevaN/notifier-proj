# Notifier Service (gRPC) 🚀

Микросервис уведомлений на Go, работающий по протоколу gRPC. Входит в состав системы ToDo-приложения и отвечает за отправку уведомлений при создании новых задач.

## Стек технологий
* **Go** (1.21+)
* **gRPC** & **Protocol Buffers**
* **Slog** (структурированное логирование)
* **Docker** & **Docker Compose**

## Как это работает
Сервис принимает запросы от основного приложения (`todo-app`) через gRPC. При получении уведомления он логирует данные (в реальной системе здесь могла бы быть отправка в Telegram, Email или SMS).

Взаимодействие описано в контракте `api/proto/notifier.proto`.
Сервир реализует интерфейс, сгенерированный с помощью `protoc`.

## Ключевые особенности
- **Слабая связанность:** Сервис полностью незивисим от ToDo-сервиса.
- **Высокая производительность**: Использование gRPC обеспечивает эффективную бинарную передачу данных.

## Быстрый запуск (Docker)
Сервис подготовлен к запуску в контейнере:

```bash
docker-compose up --build
```

Сервис будет доступен на порту :50051.

## Контракт (gRPC)

Метод SendNotification принимает: 
- task_title: заголовок задачи
- message: текст уведомления
